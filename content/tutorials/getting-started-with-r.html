Setting up R and RStudio properly is a good first start to programming in R, and learning to use GitHub as a version control tool from the beginning is a way to begin developing your portfolio track progress, and share your work publicly with others in the rstats community.

This post is intended for anyone who is interested in learning the language, but hasn’t yet installed the requisite R shell or GUI.

R and R Studio - What’s the difference?

R (the application) is a free software programming language, and a software environemnt for statistical computing and graphics which uses personal computer resources (e.g. RAM) to process the R programming language.

RStudio is a free and open-source integrated development environment (IDE), a software application that provides tools for programming in R and other languages. RStudio is the industry standard and is highly recommended for use.

Downloading and Installing R

R can be downloaded from cran by selecting the cran mirror closest to you, you can find the latest version here - make sure you download the latest version (Feather Spray). I recommend using the default installation settings, which can be changed later if desired.

Downloading and Installing RStudio

Customizing RStudio

Customization in RStudio has some components which I refer to as being “preferential” or “highly recommended”. The latter are based upon my own experience and general best practices in programming in R to get into the habit of early-on. For a detailed list of all of the rstudio settings, check out the RStudio page on customizing RStudio.

Appearances (preferential)

In RStudio Windows, select View -> Panes -> Pane Layout…

Which will bring up the following screen:

From here, in the Appearance tab, you can select the font and background colour of your preference. As you program more, you’ll see which fonts are best suited for you, remember to select based on clarity and style: you’ll be looking at the text for a long time! It is generally recommended to use a large font (e.g. size 16), this will put less strain on your eyes!

Pane Layout (preferential)

Explore a little with the pane layout, you can keep the default (console and terminal at the bottom), or move them to the upper right. I have the latter setting in place, I find it opens up more space for the script and viewer.

Workspace settings (HIGHLY recommended)

DO NOT SAVE .RDATA when you quit, and DO NOT LOAD .RDATA when you fire R up. This is a default setting in RStudio which means that whenever you do something in one sesssion, it automatically saves your work to a temporary data file, and loads it when you begin.
